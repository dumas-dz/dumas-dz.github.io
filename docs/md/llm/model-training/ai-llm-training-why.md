## 为什么要学

- 通用大模型是"博学家"非领域专家，垂类模型（医疗/法律/农业等）缺口巨大
- 大模型内部机理存在大量开放问题，是科研创新的绝佳试验场
- 企业基于私有数据训练专属模型，是数字化转型的核心竞争力

## 训练 vs RAG

| 维度 | 大模型训练 | RAG |
|------|----------|-----|
| 本质 | 知识内化于模型参数 | 推理时外挂知识库 |
| 精度 | 处理复杂/隐含逻辑查询更精准 | 受限于检索片段质量 |
| 速度 | 无需外部检索，响应更快 | 有检索耗时 |
| 可靠性 | 内置知识作后备，增强鲁棒性 | 检索失败则质量下降 |
| 实时性 | 更新需重新训练，成本高 | 动态信息优势不可替代 |

> 最佳实践：训练的"内化知识" + RAG的"外部扩展"相结合。

## 大模型训练完整流程

```
原始数据 → 清洗 → 预训练(学语言) → SFT(学对话) → 强化学习(贴近人类)
```

1. **预训练**：海量无标注文本学习语法、常识、推理（"读万卷书"）
2. **监督微调 SFT**：高质量对话数据教会指令理解和结构化表达（"名师指导"）
3. **强化学习对齐 RLHF**：根据人类偏好纠正输出，更自然安全有用（"实战打磨"）

> 流程是**评估→反馈→迭代**的循环，非单向直线。DeepSeek-R1 等模型证明强化学习是赋予模型思维与推理能力的核心路径。
